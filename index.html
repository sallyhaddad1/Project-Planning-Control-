PROJECT PLANNING - CONCEPTS AND PRINCIPALS

TARGET

What is a “Project”?

A Project is “a temporary endeavor undertaken to create a unique product, service, or result.”

Attributes of projects:
1. Unique purpose 2. Temporary 3. Require resources 4. Should have a primary sponsor and/or customer 5. Involve uncertainty

What is “Planning”?


Planning is a basic management function, it’s the process of setting goals, developing strategies, and outlining tasks and schedules to accomplish the goals.

Categories of Planning

What is “Project Planning”?

Project planning is walking through the project and getting it organized before it’s actually done.

Principals of Project Planning
1. The preliminary plans must finish before the start of execution. "A good plan today is better than a perfect plan tomorrow!".
2. The planning is a progress elaborated process (rolling planning).
3. Involve project personnel responsible for the work in creating the project schedule.
4. Include all aspects of the project such as scope, budget, schedule and quality in the planning process.
5. The plan should be realistic and should take the risk into consideration without padding.
6. Maintain a continuous flow of work with no idle periods.
7. Try to develop a flexible plan by having the minimum critical paths.
8. Keep the plan simple and summarized as possible.
9. Distribute the plan to all parties involved in the project.
10. The plan should be officially approved from all parties.
11. The plan is not expected to be precisely correct (80% is 100%).


“Failing to plan is planning to fail!".

1. Develop Project Management Plan

The Project Management Plan is a document that contains the information for how the project will be planned, executed, monitored and controlled, and closed.

Schedule Management Plan

(some times called Planning Execution Plan) is a part of the Project Management Plan.
It could be formal (included in the contract or requested from the contractor), or informal.

The purpose of this document is to illustrate the planning and follow-up deliverables and techniques. It includes information like:
- The scheduling methodology and scheduling software to be used on the project.
- Types and frequency of reports.
- The performance measures that will be used, the calculations, and the weighting factors (effort, cost, or price).
- How schedule variances will be managed.
- Schedule change control procedure.

2. Collect Requirements


Defining what is and is not included in the project. Including both:

Product scope (the features and functions of the product of the project) and

Project scope (the work you need to do to deliver the product of the project such as meetings, reports, analysis, etc.)

3. Define Scope

Study the contract in order to identify the following:
** The obligations of each party (Responsibility Assignment Matrix).
** The project major constraints and milestones and any subjected liquidated damages or bonuses.
** Related approved schedules, if any, such as the interface contractors' schedules or the nominated subcontractors' schedules in order to avoid any conflict in dates.

In this phase we start documenting the Constraints and Assumptions.

4. Create WBS

Work Breakdown Structure

- A graphical of the hierarchy of the project.
- WBS enables the clear definition and communication of project scope and performs a critical role as a monitoring and controlling tool.

Create WBS…


  • • Scope Definition: WBS identifies all work; if it's not in the WBS, it's not part of the project.
  • • Match the contractual WBS; decompose by type, discipline, phase, or location.
  • • Similar project WBS can be used as a reliable basis.
  • • Complete higher levels before breaking down further.
  • • The 100% Rule: Sum of elements at each level must represent 100% of the parent element.
  • • The entire project is included in the highest levels; some branches may be deeper than others.
  • • Lowest level is the Work Package; level above is the Control Account.
  • • Elements should be named using Nouns rather than Actions.
  • • Cost and time are estimated at the Work Package or activity level.
  • • The WBS Dictionary provides detailed descriptions for each Work Package.
  • • The WBS development process is iterative.
  • • Code of Accounts:A numbering system to identify each WBS item uniquely.
  • • Once completed, data should be entered into scheduling software.

8. Estimate Activity Durations…

How is estimating done?

- Analogous Estimate(Top-down): can be done for a project or for an activity. It’s a form of Expert Judgment.

- Parametric Estimating: uses mathematical model to calculate the projected time
*Results of parametric estimates can become heuristics.

-One-time Estimate:
*one estimate per activity is received!

- Three-point Estimates: (Program Evaluation & Review Technique – PERT analysis)
Expected Activity Duration = (P+4M+O)/6
Where: O = Optimistic, P = Pessimistic, M = Most Likely .

8. Estimate Activity Durations…

Using Standard Norms

Example:

Estimate the duration required to form 1500 m2 of RC panels if you know that: according to the norms the crew productivity is 100 m2/day. the maximum capacity is 2 crews.

Simply…

Duration = Quantity/Productivity Rate = 1500 / (100 x 2)
= 7.5 ≈ 8 working days

Schedule Constraints

Start on or after

- Prevents the schedule activity from being scheduled to start earlier than an imposed date.
- It impacts the early dates only if the imposed date is later than the network calculated early start.

Start on or before

- Prevents the schedule activity from being scheduled to start later than an imposed date.
- It impacts the late dates only if the imposed date is earlier than the network calculated late start.

Start on

Risk Management…


Example:

Through the studies to prepare a financial offer, we found out that the current cost of cement is 500 LE/ton and there is a probability of 10% that it would increase to 700 LE/ton during the project execution period.

As a respond of this risk, we will implement one of the following alternatives:
- Transfer the risk by adding a binding formula to increase the prices upon any increment in cement cost. Or
- Add a Contingency Reserve due to predicted increment in cost impact x probability= 200 x 10% = 20 LE/ton.

MONITORING & CONTROLLING

The schedule control consists of measuring the performance against the plan, and taking preventive and corrective actions during the life of the project.

We usually use periodically and non-periodically performance reports and meetings to assist controlling the project.

PERFORMANCE MEASUREMENT & FOLLOW-UP


The progress measurement reports should comply with the “Schedule Management Plan” and may include:

- Narrative Report
- Progress Sheet (Control Sheet / Spread Sheet)
- Updated Schedules
- Look-ahead Schedules
- Workforce Status
- Earned Value Measurements
- Graphical Reports

Narrative Report

This report is developed using compositional sentences and summary tables . It should be simple, summarized and comprehensive as possible; and should contain information such as:

1. Brief description of the project scope of work and key information.
2. Critical milestones planned and actual dates.
3. Total planned and actual performed quantities.
4. Scope of work, status report, and progress report for each main discipline.
5. Delay analysis and corrective actions.
6. Planned activities in the next period.
7. Total consumption of manpower and major equipment.
8. Area of concern.
9. Pictures.

Narrative Report

Progress Sheet

Progress Sheet (also called Control Sheet) is a tabular report that measure the progress of the project by comparing between planned and actual status up-to-date.

This sheet is the base of most of other reports, and its mainly outputs are:
* Activities and WBS elements planned and actual percentage completion.
* Each activity scheduled and actual performed quantities.

Remember:
The Progress Sheets are useful tools to illustrate the deviation in progress, but to estimate the impact of this deviation on the project we need to develop an Updated Schedule.

Narrative Report

Progress Sheet

  1. Project title and Schedule Baseline revision number
  2. Data Date (DD)
  3. WBS element description

  5. Early Start (ES) and Early Finish (EF)
